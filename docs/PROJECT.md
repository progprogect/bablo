# Bablo — личный торговый терминал для BingX

## Что это

Минималистичное приложение-прослойка над BingX (USDT-M Perpetual Futures) для одного трейдера.
Главная цель — сделать процесс открытия сделки максимально быстрым и **автоматически принудить к соблюдению риск-плана**: приложение само блокирует сделки, нарушающие план, и защищает от тильта.

- Деплой: Railway (один сервис + PostgreSQL).
- Клиент: PWA, устанавливается на iPhone через «Добавить на экран Домой» (без подписи, нужен только manifest).
- Пользователь: один (личное использование), вход по PIN.

## Ценность

| Проблема | Решение в Bablo |
|---|---|
| Терминал биржи перегружен, открытие сделки — много кликов | Один экран, 4 актива табами, 2 поля, 2 кнопки |
| Нарушение риск-плана «на эмоциях» | Риск-движок проверяет каждую сделку до отправки на биржу |
| Тильт после убытков / эйфория после прибыли | Автоблокировки кнопки открытия по дневным лимитам |
| Нет мотивации следовать плану долго | Геймификация: визуальное дерево роста уровней риска |
| Нет данных для анализа своих сделок | Автотрекинг каждой сделки (MFE, безубыток, сигналы) |

## Экраны (всего 3 + навигация нижними табами)

### 1. Главный экран (дашборд) — единственный «рабочий» экран

- Сумма депозита (баланс фьючерсного аккаунта).
- Индикатор текущего уровня риска и прогресса (компактный, ведёт на дерево роста).
- Табы активов: **TIA · TAO · VIRTUAL · WLD** (набор редактируется в админке).
- Форма открытия сделки (см. флоу ниже).
- Карточка активной сделки (если есть): актив, лонг/шорт, цена входа, SL, TP,
  сумма риска, R/R, плечо, цена ликвидации, текущий PnL.
- Если действует блокировка — кнопки открытия заблокированы, показана причина и таймер до разблокировки.

### 2. История (2 вкладки внутри экрана)

**Вкладка «Сделки»:**
- Карточка-подсказка над списком:
  - «Самые прибыльные часы» — часы открытия, в которые **больше половины** открытых в этот час
    сделок дошли до тейка (не просто `resultR > 0` у отдельной сделки — час считается прибыльным
    по доле тейков среди сделок, открытых в нём). Формат: `9-10ч — 4/5 TP` (открыто 5 сделок,
    4 из них дошли до тейка). Топ-3 по числу тейков.
  - «Самый прибыльный актив» — по сумме $, формат `TAO — 1/2 TP (50%)`: сколько из ВСЕХ сделок
    по этому активу дошло до тейка.
  - Статистика по пресетам R/R — по каждому пресету отдельной строкой: сколько сделок реально
    дошло до тейка (`tpCount/totalTrades`, %) и, если были промахи, сколько из них закрылось по
    стопу и с каким средним R именно у стоповых (не смешивая с прибыльными исходами — иначе
    среднее по маленькой выборке выглядит как случайное число без контекста); список раскрывается
    по кнопке, если пресетов много.
  - Час, в который чаще всего открывались сделки, закрытые по стопу, и типичный час достижения
    дневной цели +NR.
- Карточки сделок: актив, лонг/шорт, дата/время открытия, продолжительность, R/R, реализованный
  результат в USDT (со знаком, зелёным/красным). Причина закрытия показана только для SL/TP —
  для ручного закрытия/обнаруженного постфактум на бирже отдельная подпись не нужна.

**Вкладка «Статистика»:** карточка на каждый месяц (сначала новые) —
% результата к депозиту, сумма R, всего сделок, разбивка TP/SL/безубыток, винрейт, число
торговых дней, разбивка закрытых по тейку по всем пресетам R/R (подписаны множителем —
«1R, 1.5R, 2R…», нулевые пресеты тоже показаны) и рядом с ней — суммарный R отдельно по
прибыльным и по убыточным сделкам за месяц («+R / −R»). Кнопка «График роста депозита»
открывает полноэкранный шит с кривой линией по снимкам эквити (**осознанное исключение**
из принципа «без графиков» ниже, подтверждено пользователем 14.07.2026) — растёт только
вперёд, с момента появления первого снимка, без восстановления прошлых точек.

% к депозиту считается не по снимку ровно на начало каждого месяца (таблица снимков
эквити молодая, для прошлых месяцев такого снимка физически нет), а восстановлением
"в обратную сторону" от последнего известного снимка: берём текущий баланс и вычитаем
из него реализованный PnL всех сделок, закрытых с начала нужного месяца, а также ручные
пополнения/выводы за тот же период (`equity_adjustments`, задаются в админке — иначе
любое пополнение исказило бы % за все месяцы до него). Так процент доступен для ВСЕЙ
истории сделок, а не только для месяцев после появления снимков.

### 3. Админка

- BingX API-ключи (хранятся зашифрованными на сервере).
- Управление активами: символ, плечо (TIA 20x, TAO 25x, VIRTUAL 25x, WLD 20x), добавление/замена.
- Кастомизация параметров риск-плана: лестница уровней, дневные лимиты, время сброса дня (по умолчанию 07:00 UTC+3).
- Пополнения/выводы средств (дата, сумма, заметка) — для корректного % к депозиту в «Статистике» (см. выше).
- «Пересверить закрытые сделки» — повторно сверяет с BingX сделки, закрытые как «внешние»
  (`closeReason = "external"`), и чинит причину закрытия (SL/TP) там, где BingX ещё хранит
  данные по ордеру. Нужна из-за бага в `getOrderStatus` до 16.07.2026 (см. ARCHITECTURE.md).
- «Очистить данные для нового аккаунта» — полный сброс истории/прогресса перед подключением
  другого BingX-аккаунта (недоступно при активной сделке).

## Флоу «Открыть сделку» (двухшаговый)

**Шаг 1 — вход в позицию (SL обязателен до входа):**
1. Выбираю актив табом.
2. Заполняю «Объём позиции» (количество монет) и «Цена SL».
3. Нажимаю **Купить** или **Продать**:
   - риск-движок считает точный риск: `|текущая цена − SL| × количество`;
   - если риск выходит за пределы ±20% от 1R текущего уровня (больше ИЛИ меньше) — поля
     подсвечиваются красным, сделка не отправляется;
   - если ок — рыночный ордер (изолированная маржа, one-way, плечо из настроек актива)
     и **сразу** выставляется SL-ордер.

**Шаг 2 — тейк-профит:**
4. Заполняю «Цена TP» вручную, либо выбираю пресет R/R: `1/1 · 1/1.5 · 1/2 · 1/3 · 1/4 · 1/5 · 1/6 · 1/7`
   (TP рассчитывается от фактической цены входа и SL).
5. Нажимаю **Сохранить** — на бирже выставляется TP, показывается карточка активной сделки.

## Ограничения (риск-движок)

Кнопка «Купить/Продать» блокируется, если:

| Условие | Блокировка |
|---|---|
| Закрылась любая сделка (кулдаун, независимо от результата) | на 1 час |
| За день зафиксировано −2R от депозита | до 07:00 следующего дня |
| За день зафиксировано +3R | до 07:00 следующего дня |
| За день 2 и более сделки закрыты по стопу (независимо от порядка и суммы R) | до 07:00 следующего дня |
| Уже есть активная сделка в приложении | пока сделка не закрыта |
| На BingX открыта позиция не через приложение | пока позиция не закрыта на бирже |

При блокировке UI показывает понятное пояснение причины и таймер до разблокировки. Позиции,
открытые вручную на бирже (не через приложение), показываются на дашборде отдельным
read-only блоком (без SL/TP/риска — они неизвестны приложению) вместо формы открытия сделки.

Важно: блокировки действуют **только на открытие новых сделок**. Активная сделка никогда
не закрывается системой принудительно — она дорабатывает до своего SL/TP или ручного
закрытия, и лишь после этого её результат учитывается в дневных лимитах.

## Принципы продукта

1. **Минимализм**: один главный экран, остальные — только при необходимости. Никаких графиков, стаканов, лент сделок.
   Единственное осознанное исключение — график роста депозита на вкладке «Статистика»
   (простая линия по дневным снимкам эквити, без внешних библиотек), добавлено по
   явному запросу пользователя 14.07.2026.
2. **Дисциплина важнее свободы**: приложение не даёт нарушить план — это фича, а не ограничение.
3. **Скорость**: открытие сделки — не более 4 действий.
4. **Без поллинга**: realtime через WebSocket BingX и SSE к клиенту.
